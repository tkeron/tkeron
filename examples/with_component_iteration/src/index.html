This example demonstrates how Tkeron's build process iterates through components:

Step 1: Dynamic Data Processing

The <card-list> component is a .com.ts that:

  • Processes user data with TypeScript logic
  • Calculates engagement scores and status
  • Generates <user-card> components with computed attributes

How It Works - Mixed .com.ts and .com.html

  1. Iteration 1 (.com.ts): card-list.com.ts performs calculations (engagement scores, time deltas) and generates <user-card> elements with data
  2. Iteration 2 (.com.ts): user-card.com.ts reads attributes, formats role labels, and inserts <status-badge> and <engagement-meter> with specific data
  3. Iteration 3 (.com.ts + .com.html):
    • status-badge.com.ts applies conditional logic for colors/icons based on status
    • engagement-meter.com.ts calculates percentages and levels with TS logic
    • note-box.com.html provides static HTML wrapper (no logic needed)
  4. Iteration 4: No more components found, build completes!

Key differences:

  • .com.ts = Dynamic processing, calculations, conditional logic, data transformation
  • .com.html = Static templates when no logic is needed, just consistent markup

Note: Custom elements must have a hyphen (like <my-component>).
